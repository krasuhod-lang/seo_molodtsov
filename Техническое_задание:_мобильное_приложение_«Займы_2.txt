Техническое задание: мобильное приложение «Займы 24/7»
1. Общая информация
Необходимо разработать мобильное приложение-каталог Займы 24/7, которое представляет собой витрину CPA-офферов микрофинансовых организаций. Приложение не выдает займы, а только отображает список офферов и перенаправляет пользователя по внешней ссылке на оффер. Визуальная структура и стиль должны соответствовать приложенному макету из Figma.

2. Цель приложения
Основная цель приложения — показать пользователю подборку займов и повысить конверсию в переходы по CPA-ссылкам. Для этого в приложении должны быть:

onboarding-слайдер при первом запуске;

список CPA-офферов с сортировкой;

калькулятор займа;

страница контактов;

нижнее меню навигации;

административная часть для управления контентом и офферами.

3. Пользовательские сценарии
Первый запуск
При первом запуске приложения после splash-экрана показывается onboarding-слайдер из 3 экранов. На каждом экране есть кнопки «далее» и «пропустить». Кнопка «далее» переключает на следующий слайд, кнопка «пропустить» закрывает все слайды и переводит в основное приложение.

Повторный запуск
При повторном запуске приложения onboarding-слайдер не показывается. Пользователь сразу попадает в основную часть приложения после splash-экрана.

Навигация
В приложении должно быть нижнее меню, которое отображается на всех основных страницах, кроме заглушки и onboarding-слайдов. Вверху всех страниц должна быть стандартная плашка с возможностью вернуться на главную страницу и названием текущего раздела. На главной странице вместо названия раздела отображается название приложения Займы 24/7.

4. Экраны приложения
4.1 Splash screen
При запуске приложения сначала отображается splash-экран. Дизайн splash должен соответствовать общему стилю приложения и макету Figma.

4.2 Onboarding-слайдер
Слайдер показывается только при первом запуске.

Слайд 1

Заголовок: Деньги онлайн на карту

Текст: Оформите заявку на займ 3-5 компаниях и получите максимальный шанс одобрения

Слайд 2

Заголовок: Первый займ бесплатно

Текст: Для новых клиентов компаний-партнеров

Слайд 3

Заголовок: На связи 24/7

Текст: Заявки рассматриваются круглосуточно, без выходных

На каждом слайде:

кнопка «далее»;

кнопка «пропустить».

4.3 Главная страница
Главная страница — это витрина CPA-офферов, выстроенных в заданном порядке.

Заголовок: Займы 24/7
Подзаголовок: На вашу банковскую карту, с любой кредитной историей

Поведение списка:

по умолчанию отображаются первые 6 офферов;

после 6-го оффера отображается кнопка «Показать еще»;

при нажатии показываются следующие 6 офферов;

если офферов меньше 6, отображаются все доступные;

офферы должны отображаться в порядке, заданном через админку.

Карточка оффера должна содержать:

логотип МФО;

название МФО;

слоган;

сумму займа;

срок займа;

возраст;

кнопку «Получить деньги», ведущую на ссылку оффера.

4.4 Страница «Рейтинг МФО»
Страница дублирует функционал главной страницы, но с другим текстовым оформлением.

Заголовок: Рейтинг МФО
Подзаголовок: Рейтинг составлен на основе отзывов из открытых источников

Функционально страница должна работать так же, как главная: список офферов, пагинация/дозагрузка по 6 элементов, сортировка, переход по ссылке оффера.

4.5 Страница «Калькулятор»
На странице калькулятора должны быть:

ползунок выбора суммы;

ползунок выбора срока;

чекбокс «Первый займ бесплатно»;

блок с результатами расчета.

Параметры:

сумма: от 0 до 100 000;

срок: от 0 до 365 дней;

ставка: 0,8% в день;

при включении чекбокса «Первый займ бесплатно» ставка становится 0% в день.

Блок вывода:

Сумма займа = выбранная сумма;

Срок займа = выбранный срок;

К возврату = результат расчета.

Формула расчета:

без акции: К возврату = Сумма + (Сумма × 0,8% × Срок)

с акцией: К возврату = Сумма

4.6 Страница «Контакты»
На странице вывести следующий текст:

По всем вопросам, связанным с работой приложения, можно связаться с нами по e-mail:
avada-gorup@yandex.ru
Сервис не предоставляет займы и не является кредитной организацией. Приложение выполняет функцию каталога микрофинансовых организаций, предоставляющих информацию об услугах. Вся информация о займах и организациях взята из открытых источников и предоставляется исключительно в ознакомительных целях.

5. Административная часть
Необходимо реализовать админку для управления контентом приложения.

5.1 Управление страницами
Администратор должен иметь возможность:

изменять тексты на страницах;

редактировать заголовки и подзаголовки;

при необходимости менять текст onboarding-слайдов;

редактировать текст страницы контактов.

5.2 Управление CPA-офферами
Администратор должен иметь возможность:

добавлять неограниченное количество офферов;

редактировать существующие офферы;

удалять офферы;

менять порядок отображения офферов вручную.

5.3 Поля оффера
Для каждого CPA-оффера должны быть доступны поля:

Название МФО — текстовое поле;

Сумма — числовое поле, справа отображается символ ₽;

Срок — числовое поле, справа отображается дн.;

Возраст — текстовое поле;

Слоган — текстовое поле, отображается под названием и логотипом, например: «Первый займ бесплатно»;

Логотип — загрузка изображения;

Ссылка на оффер — URL-поле, используется для кнопки «Получить деньги».

5.4 Сортировка
Должна быть возможность вручную задавать порядок офферов:

какой оффер отображается первым;

какой вторым;

и так далее.

Порядок должен сохраняться и использоваться на главной странице и на странице рейтинга МФО.

6. Требования к интерфейсу
Дизайн должен соответствовать макету Figma.

Интерфейс должен быть адаптирован под мобильные устройства.

Основные действия должны быть доступны в 1–2 касания.

Навигация должна быть простой и понятной.

Карточки офферов должны быть визуально одинаковыми и легко масштабироваться при добавлении новых элементов.

7. Поведение приложения
Splash показывается при каждом запуске.

Onboarding показывается только один раз.

Нижнее меню отображается на основных экранах.

Кнопка «Получить деньги» открывает внешнюю ссылку оффера.

Админка должна обеспечивать простое добавление нового контента без правки кода.

8. Что нужно учесть в реализации
Хранение признака первого запуска приложения.

Возможность динамической подгрузки офферов из источника данных.

Возможность сортировки офферов через административный интерфейс.

Поддержка пустого состояния, если офферы не загружены.

Валидирование полей админки.

Безопасная обработка внешних ссылок.
